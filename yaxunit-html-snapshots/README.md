# Снимки HTML для макетов YAXUnit

Снимки пишутся **относительно `projectPath`** в настройках YAXUnit:

**`<projectPath>/yaxunit-html-snapshots/test_templates/ИмяМакета.snapshot.txt`**

В этом репозитории **`projectPath`** в YAXUnit задаётся как **корень `Kanban_for_1C`**; тогда снимки в дереве git — **`yaxunit-html-snapshots/test_templates/`** в корне репо. Скрипт **`apply-yaxunit-html-snapshots.js`** читает **только** этот каталог.

При **`ПроверитьТекстHTMLКанбанДоски`** файл **`ИмяМакета.snapshot.txt`** создаётся только если его **ещё нет** в текущем прогоне; уже созданный в этом прогоне файл **не перезаписывается** (второй вызов с тем же именем макета не подменяет снимок).

Перед клиентской фазой модуля **`канбан_Тесты`** в **`ПередВсемиТестами`** (до серверной подготовки данных) вызывается **`канбан_ТестыВызовСервера.ОчиститьКаталогСнимковHTMLДляМакетов()`** — каталог **`test_templates`** очищается от **`*.snapshot.txt`** прошлого прогона; затем в ходе тестов файлы создаются заново по мере первых проверок.

Перечисленные ветки **`yaxunit-html-snapshots/`** с снимками в git **не коммитятся** по умолчанию (см. корневой `.gitignore`). В корне репозитория этот каталог может содержать только этот **`README.md`**.

## Перенос в макеты EDT

**`OneKanban-html/scripts/apply-yaxunit-html-snapshots.js`** берёт файлы из **`Kanban_for_1C/yaxunit-html-snapshots/test_templates/`** (корень репозитория рядом с **`OneKanban-html`**).

```bash
cd OneKanban-html
npm run yaxunit:apply-snapshots
```

Имя **`yaxunit:apply-snapshots`** — ключ в **`package.json` → `scripts`**, выполняется **`node scripts/apply-yaxunit-html-snapshots.js`**.

### После переноса в макеты (EDT)

Файлы **`Template.txt`** обновляются на диске. Если **1C:EDT** уже открыт с проектом **`Управление_задачами.YAXUNIT`**, среда может не подхватить изменения сразу (кэш редактора, открытые вкладки макетов).

Рекомендуется одно из действий:

- **Обновить ресурсы** в навигаторе: контекстное меню на проекте **`Управление_задачами.YAXUNIT`** или на **`CommonTemplates`** → *Обновить* (аналог **F5** в Eclipse).
- Либо **закрыть и снова открыть** проект / перезапустить EDT, если содержимое макетов в дереве по-прежнему старое.

Скрипт **не** закрывает EDT и **не** управляет IDE — только перезаписывает файлы в репозитории.

Подробнее: **Разработка → Сборка OneKanban-html** (`documentation/docs/Разработка/Сборка-OneKanban-html.md`).
