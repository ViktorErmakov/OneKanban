# Скриншоты для пользовательской документации

Каталог: `documentation/static/img/`

## Снимки с HTML-доски (демо / Playwright)

Сгенерировать автоматически из собранного `OneKanban-html`:

```bash
cd OneKanban-html
npm run build
cd ../documentation
node scripts/capture-board-screenshots.mjs
```

Скрипт создаёт или обновляет:

| Файл | Содержание |
|------|------------|
| `CreateKindModal.png` | Модалка «Что создать?» у кнопки «+» |
| `CardAnatomy.png` | Карточки задачи и ошибки |
| `MaxVisibleProjects.png` | Выбор числа видимых пилюль 1–5 |
| `ExecutorNoAssignee.png` | Фильтр исполнителей с «Без исполнителя» |
| `UrgencyHexRgb.png` | Поповер срочности с HEX/RGB |
| `UrgencyResetAll.png` | Меню срочности, «Сбросить настройки» |
| `DnDForbidden.png` | Группировка по проектам, запрет DnD |
| `HiddenFilters.png` | Доска только с задачами, скрыт фильтр «Тип» |
| `NoExecutorPhoto.png` | Доска без фото исполнителей |

## Снимки из 1С (нужно сделать вручную)

Сохранить PNG в этот каталог с указанными именами. Рекомендуемая ширина: **1200–1400 px**, светлая тема.

| Файл | Что снять |
|------|-----------|
| `NoSettings.png` | Доска при первом запуске без настроенных статусов |
| `status.png` | Форма настроек, вкладка **Статусы** |
| `ОпределениеКнопкиДобавить.png` | Таблица статусов, колонка **Кнопка добавить задачу** |
| `project.png` | Вкладка **Проекты** с цветами |
| `comparisonSetting.png` | Включение опции **Сопоставление статусов** |
| `comparison.png` | Вкладка сопоставления статусов |
| `Commands.png` | Раздел интерфейса **OneKanban** (доска, справочники, отчёт) |
| `PersonalSettings.png` | Блок **Персональные настройки**, галочка фото |
| `CardLinkOpen.png` | Клик по «Задача №…» → открыта форма элемента |
| `ReportHistory.png` | Отчёт **История задач**, вариант «Детально» |
| `ReportHistorySummary.png` | Сводный вариант отчёта |

### Подсказки для съёмки

- **CreateKindModal** (если снимаете в 1С): на доске должны быть и задачи, и ошибки; фильтр «Тип» сброшен; нажать «+».
- **Commands.png**: подсистема `OneKanban` в панели разделов 1С.
- Уже существующие файлы (`ControlPanel.png`, `Add.png`, …) сверьте с UI 3.0 и переснимите при расхождении.

## Проверка

```bash
cd documentation
npm run build
```

После сборки убедитесь, что в `build/` нет битых ссылок на изображения.
