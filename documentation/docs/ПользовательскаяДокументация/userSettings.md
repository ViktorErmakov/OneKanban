# Настройки доски

Форма **Настройки пользователя OneKanban** открывается кнопкой **Настройки** на панели доски. В ней сочетаются **общие** параметры (статусы, проекты, сопоставление — влияют на всех пользователей доски) и **персональные** (сохраняются отдельно для каждого пользователя).

После **Записать** или **Записать и закрыть** доска перестраивается с учётом новых параметров.

## Персональные настройки

В верхней части формы расположен блок **Персональные настройки**.

![Персональные настройки](/img/PersonalSettings.png)

### Отображать фотографию исполнителя

Если галочка **включена**, на карточках и в заголовках группировки по исполнителю показываются фотографии. Если **выключена**, фото не попадают в HTML доски — остаются только имена и фильтры по исполнителям.

![Доска c фото исполнителей](/img/ExecutorPhoto.png)
![Доска без фото исполнителей](/img/NoExecutorPhoto.png)

:::info

Персональная настройка сохраняется для текущего пользователя и не меняет отображение у других.

:::

## Статусы

Для работы OneKanban необходимо выбрать несколько **статусов** — это колонки канбан-доски.

Порядок расположения статусов на доске соответствует значению в колонке **Номер по порядку**.

![Статусы](/img/status.png)

Последняя колонка таблицы — **Кнопка добавить задачу**.

![Установка кнопки добавить задачу](/img/ОпределениеКнопкиДобавить.png)

![Кнопка добавить на доске](/img/Add.png)

Если установить галочку напротив статуса, в заголовке колонки на доске появится кнопка **+** для создания задачи или ошибки в этом статусе. Подробнее — в разделе [Добавление задачи или ошибки](./OneKanban.md#добавление-задачи-или-ошибки).

## Глобальные настройки

### Отключить отправку статистики

По умолчанию при каждом открытии канбан-доски **сервер 1С** отправляет один HTTP-запрос в Яндекс.Метрику (`mc.yandex.ru`). Это не браузерный счётчик.

- **Куда:** `mc.yandex.ru` (протокол Measurement Protocol).
- **Что:** факт открытия доски; числовой псевдоним «пользователь в этой базе» (ClientID из цифр GUID информационной базы и пользователя 1С); служебные параметры счётчика.
- **Зачем:** оценить, сколько баз и пользователей пользуются расширением, и показывать агрегированную статистику на сайте проекта.
- **Чего нет:** имён, логинов, задач, проектов, статусов, настроек и любых бизнес-данных.

Чтобы **отключить** отправку для всей базы, включите переключатель **Отключить отправку статистики** на вкладке **Глобальные**.
Глобальная настройка влияет на всех пользователей. Альтернатива для администраторов — блокировка исходящих соединений к `mc.yandex.ru` на уровне сети.

## Сопоставление статусов

:::info

Данный функционал создавался для СППР, но можно использовать и в других конфигурациях при необходимости.

:::

![Сопоставление — флаг](/img/comparisonSetting.png)

При включении функциональной опции **Сопоставление статусов** появляется одноимённая вкладка.

![Сопоставление статусов](/img/comparison.png)

В ней необходимо сопоставить статусы задач и статусы ошибок. Тогда на канбан-доске в колонках статусов задач будут отображаться и ошибки (с красной полосой на карточке).

![Ошибки и задачи на доске](/img/Errors.png)

## Команды подсистемы OneKanban

В разделе интерфейса 1С **OneKanban** доступны команды для работы вне доски:

![Команды подсистемы](/img/Commands.png)

| Команда | Назначение |
|---------|------------|
| **Канбан-доска** | Открытие HTML-доски |
| **Задачи** | Список справочника задач |
| **Ошибки** | Список справочника ошибок (при использовании сопоставления) |
| **Проекты** | Справочник проектов |
| **Статусы задач** / **Статусы ошибок** | Справочники статусов |
| **Сопоставление статусов** | Регистр пар «статус задачи ↔ статус ошибки» |
| **История задач** | Отчёт по времени в статусах |

Подробнее о справочниках и отчёте — на странице **[Справочники и отчёты](./CatalogsAndReports.md)**.
